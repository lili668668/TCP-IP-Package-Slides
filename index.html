IP 協定的主要兩個功能

  • 功能一：裝箱
  • 功能二：找路

功能一：裝箱

需要看 MTU（Maximum Transmission Unit 最大傳輸單位）

每個 Data layer 可處理的最大傳輸單位是不同的

功能二：找路

IP 封包（IP Package）

  • IP 表頭 → 投遞貨物的資料
  • IP Payload → 貨物

IPv4 表頭

  • Version
  • Header Length
  • Differentiated Services Field → 封包怎麼處理

Differentiated Services Field

DSCP → 優先級

DSCP 名稱數值用途
CS0 (Default)0一般流量，無優先級
CS18低優先級（背景流量）
AF11~AF43各種一般商務應用，分等級保證
EF (Expedited Forwarding)46即時通訊（VoIP 電話、視訊會議）
CS648網路控制協定
CS756最高優先級（保留給網路自己用）

ECN → 路上有沒有塞車

ECN 值名稱意思
00Not-ECT這個封包不支援 ECN
10ECT(0)支援 ECN（傳輸層標記法 0）
01ECT(1)支援 ECN（傳輸層標記法 1）
11CE路由器標的：我這裡塞車了！

IPv4 表頭（續）

  • Total Length → IPv4 封包總長度
  • Identification → 封包識別 ID（分片重組用）
  • Flags：Reserved bit / Don't fragment / More fragments
  • Fragment Offset → 偏移量
  • Time to Live (TTL) → 每過一個路由減一，歸零即丟棄
  • Protocol → 協議（UDP / TCP）
  • Header Checksum → 校驗碼（IPv6 已移除）
  • Source / Destination Address

IPv6 表頭

  • Version
  • Traffic Class → 同 IPv4 的 Differentiated Services Field
  • Payload Length → 資料長度（不含表頭）
  • Next Header → 下一個協議
  • Hop Limit → 同 IPv4 的 TTL
  • Source / Destination Address
  • ✓ Checksum 消失了 owo

ARP 協議

Address Resolution Protocol

IP → MAC 地址 的轉換協議

ARP 封包

  • Hardware type → 網路技術
  • Protocol type → 協議
  • Hardware size → MAC address 的長度
  • Protocol size → IP address 的長度
  • Opcode → 執行代碼
  • Sender MAC address → 傳送者的 MAC address
  • Sender IP address → 傳送者的 IP address
  • Target MAC address → 接收者的 MAC address
  • Target IP address → 接收者的 IP address

DNS 封包

  • Transaction ID
  • Flags
  • Questions → 問題數
  • Answer RRs → 有幾筆回答
  • Authority RRs → 有幾個 NS 有這個記錄
  • Additional RRs → 額外的答案

有發現一件事嗎？

ARP 跟 DNS 封包沒有加密 (° ∀ °)
沒加密 = 可攻擊

DNS 汙染

[A] google.com IP 是什麼？ → [中間人] google.com IP 是什麼？ → [DNS Server]

[A] ← google.com IP 是 8.87.87.2 [中間人] ← google.com IP 是 3.12.45.7 [DNS Server]

DNS over TLS

讓 DNS 經過 TCP 連線，且加密

ARP 攻擊防範

  • Dynamic ARP Inspection → 監聽 DHCP 的交握流程，確保每個 ARP 封包資料正確
  • Private VLAN → 一個裝置一個虛擬區網，獨立隔離
  • 802.1X 連線認證 → 認證連入區網的使用者
  • 換掉 ARP → IPv6 使用 NDP 協議（ARP + ICMP）
    • 可搭配 SEND（加密版 NDP）
    • 實務上多用 RA Guard（類似 DAI 概念）

ICMP 協議

用來做網路診斷的 debug 協議

  • Type
  • Code → Type 的結果代碼（同 Type 不同 Code，意義不同）
  • Identifier → ID（可能同時執行多個 ping）
  • Sequence Number → 每個封包的 ID
  • Data → 前面是時間戳，後面是填充資料

TCP 協議

三方交握

  • 🤝 [SYN]：來，握手
  • 🤝 [SYN, ACK]：好，握手
  • 🤝 [ACK]：好，我們握手了

TCP 封包的其他部分

  • [ECE]：路上有壅塞了，請降速
  • [CWR]：收到，我已經降速了
  • [PSH]：現在立刻給我資料
  • Window Size → 告訴對方我現在腹地有多大，你可以送多大的封包過來
  • Options → 一堆參數

QUIC

Quick UDP Internet Connection

好快快 UDP

  • 我大 Google 提出的協議
  • 加密協議，資料穿好衣服
  • 基於 UDP，兼具速度與安全

QUIC 三方交握

補充：MACsec

  • 網路第二層（Data Link Layer）的連線加密防護
  • 做在第二層 → 硬體層級的加密
  • 一般家用電腦無法使用
  • 通常用於資料中心或企業網路
